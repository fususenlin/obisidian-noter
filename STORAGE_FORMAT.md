# Noter 存储格式说明

## 📁 文件存储结构

### 新的存储格式（按日期分组）

```
Vault/
└── Noter/
    └── Notes/
        ├── 2026/
        │   ├── 01/
        │   │   ├── 20260101.md    # 2026 年 1 月 1 日的所有笔记
        │   │   ├── 20260102.md    # 2026 年 1 月 2 日的所有笔记
        │   │   └── ...
        │   ├── 02/
        │   │   ├── 20260201.md
        │   │   └── ...
        │   └── 03/
        │       └── 20260301.md
        └── ...
```

### 文件路径格式
- **目录结构**: `YYYY/MM/`
- **文件名**: `YYYYMMDD.md`
- **完整路径**: `Noter/Notes/YYYY/MM/YYYYMMDD.md`

---

## 📝 笔记内容格式

### 单条笔记格式

每条笔记包含以下部分：
```
[ID] {标签 1，标签 2,...} [创建时间] 笔记内容
```

### 示例文件内容

```markdown
[20260301120000] {灵感，产品} [2026-03-01T12:00:00.000Z] 今天想到一个很好的产品创意：做一个类似 Flomo 的 Obsidian 插件

---
[20260301143000] {工作，会议} [2026-03-01T14:30:00.000Z] 下午的项目会议定在周三，需要准备好需求文档

---
[20260301180000] {读书，成长} [2026-03-01T18:00:00.000Z] 读《深度工作》有感：应该每天留出不受打扰的 2 小时专注时间
```

---

## 🔍 格式详解

### 笔记块结构

每个笔记块由以下部分组成：

1. **ID**: `[20260301120000]`
   - 14 位数字，格式：YYYYMMDDHHmmss
   - 唯一标识一条笔记

2. **标签**: `{灵感，产品}`
   - 花括号包围
   - 多个标签用逗号分隔
   - 不包含 `#` 符号

3. **创建时间**: `[2026-03-01T12:00:00.000Z]`
   - ISO 8601 格式
   - 用于排序和回顾

4. **内容**: 笔记正文
   - 可以包含多行
   - 支持 Markdown 格式
   - 自动识别 `#标签` 语法

### 分隔符

- 笔记块之间使用 `---` 分隔
- 每个 `---` 独占一行

## 💡 优势

### 1. 符合 Obsidian 习惯
- 按日期组织文件，类似日记
- 方便在文件浏览器中浏览
- 与 Obsidian 的日记功能配合良好

### 2. 减少文件数量
- 同一天的笔记在同一个文件中
- 避免创建大量小文件
- 提高文件管理效率

### 3. 便于回顾
- 打开一个文件就能看到整天的笔记
- 方便按日期回顾
- 支持按标签筛选

### 4. 版本控制友好
- 同一天的修改在同一个文件中
- Git diff 更清晰
- 便于追踪变化

## 📊 解析规则

### 正则表达式

```regex
^\[(\d{14})\]\s*\{([^}]*)\}\s*\[(\d{4}-\d{2}-\d{2}T[\d:+Z]+)\](.*?)$
```

### 匹配组

1. `(\d{14})` - ID
2. `([^}]*)` - 标签
3. `(\d{4}-\d{2}-\d{2}T[\d:+Z]+)` - 创建时间
4. `(.*?)` - 内容

---

## ⚠️ 注意事项

### 1. 标签格式
- 存储时不包含 `#` 符号
- 显示时会自动添加 `#`
- 输入时使用 `#标签` 语法

### 2. 时间格式
- 使用 ISO 8601 格式
- 包含时区信息
- 便于跨时区同步

### 3. 文件编码
- UTF-8 编码
- 支持中文和其他 Unicode 字符

### 4. 换行符
- 使用 Unix 换行符（LF）
- 跨平台兼容

## 🔧 开发参考

### 读取笔记

```typescript
const content = await app.vault.read(file);
const blocks = content.split(/\n---\n/g);

for (const block of blocks) {
  const match = block.match(/^\[(\d{14})\]\s*\{([^}]*)\}\s*\[(.*?)\](.*)$/s);
  if (match) {
    const id = match[1];
    const tags = match[2].split(',');
    const created = match[3];
    const content = match[4].trim();
  }
}
```

### 写入笔记

```typescript
const newBlock = `[${id}] {${tags.join(',')}} [${created}] ${content}\n`;
const updatedContent = existingContent 
  ? `${existingContent}---\n${newBlock}` 
  : `${newBlock}`;

await app.vault.modify(file, updatedContent);
```
