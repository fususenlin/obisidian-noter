# Noter 更新日志

## [1.1.0] - 2026-03-01

### ✨ 重大更新：新的存储格式

#### 📁 存储结构变更

**旧格式**：每条笔记一个独立文件
```
Vault/Noter/Notes/
├── 20260301120000.md
├── 20260301143000.md
└── 20260301180000.md
```

**新格式**：按日期分组存储
```
Vault/Noter/Notes/
└── 2026/
    └── 03/
        └── 20260301.md  # 包含 3 月 1 日的所有笔记
```

#### 🎯 变更详情

1. **文件路径格式**
   - 目录结构：`YYYY/MM/`
   - 文件名：`YYYYMMDD.md`
   - 完整路径：`Noter/Notes/YYYY/MM/YYYYMMDD.md`

2. **笔记内容格式**
   ```markdown
   [20260301120000] {灵感，产品} [2026-03-01T12:00:00.000Z] 笔记内容
   
   ---
   [20260301143000] {工作，会议} [2026-03-01T14:30:00.000Z] 另一条笔记
   ```

3. **元数据格式**
   - ID: `[20260301120000]` - 14 位数字标识符
   - 标签：`{标签 1，标签 2}` - 花括号包围，逗号分隔
   - 时间：`[ISO 时间]` - ISO 8601 格式
   - 分隔符：`---` - 笔记块之间的分隔线

#### 💡 优势

1. **符合 Obsidian 习惯**
   - 类似日记的组织方式
   - 方便在文件浏览器中浏览
   - 与 Obsidian 日记功能配合良好

2. **减少文件数量**
   - 同一天的笔记在同一个文件中
   - 避免创建大量小文件
   - 提高文件管理效率

3. **便于回顾**
   - 打开一个文件查看整天的笔记
   - 方便按日期回顾
   - 支持按标签筛选

4. **版本控制友好**
   - 同一天的修改在同一文件
   - Git diff 更清晰
   - 便于追踪变化

#### 🔧 技术变更

1. **NoteService 重构**
   - 新增 `getDateKey()` 方法生成日期键
   - 新增 `getFilePath()` 方法生成文件路径
   - 新增 `parseNoteBlocks()` 方法解析笔记块
   - 新增 `formatNoteBlock()` 方法格式化笔记块
   - 重写 `createNote()` 方法支持追加写入
   - 重写 `updateNote()` 方法支持按 ID 更新
   - 重写 `deleteNote()` 方法支持按 ID 删除
   - 重写 `getAllNotes()` 方法支持批量解析

2. **类型定义更新**
   - 新增 `NoteBlock` 接口
   - 更新 `Note` 接口，添加 `dateKey` 字段

3. **API 变更**
   - `updateNote()` 新增 `noteId` 参数
   - `deleteNote()` 新增 `noteId` 参数

#### 📊 性能指标

- 构建大小：30.06 KB (gzip: 7.59 KB)
- 构建时间：~255ms
- 类型检查：通过

#### 📚 文档更新

- 新增 [STORAGE_FORMAT.md](STORAGE_FORMAT.md) - 存储格式详细说明
- 更新 [README.md](README.md) - 存储格式说明
- 新增 [CHANGELOG.md](CHANGELOG.md) - 更新日志

#### ⚠️ 注意事项

1. **不向后兼容**
   - 新格式与旧格式不兼容
   - 需要手动迁移旧数据
   - 建议备份后再升级

2. **迁移建议**
   - 手动迁移旧笔记到新格式
   - 或继续使用旧版本
   - 等待官方迁移工具
